OCR Pipeline

The PDF is fully image-based (scanned). Standard text extraction returns nothing. We built a page-by-page OCR pipeline to extract clinical text.

Why page-by-page?

⚠️First attempt: converted all 71 pages at 300 DPI simultaneously → MemoryError (18GB RAM exceeded). Fixed by processing one page at a time at 200 DPI.

Tools Used

LibraryRole
pdf2imageConvert PDF pages → PIL images
pytesseractOCR engine (Tesseract 5.4)
pypdfGet page count
pdfplumberExtract tables (fallback)
💾Progress saved to JSON after every page — if it crashes, resumes from last checkpoint.

Pipeline Code

for page_num in range(1, total + 1):
    # Convert single page — low memory
    images = convert_from_path(
        PDF_PATH,
        dpi=200,
        first_page=page_num,
        last_page=page_num,
    )
    # Run Tesseract OCR
    raw_text = pytesseract.image_to_string(
        images[0], lang='eng'
    )
    del images  # free memory immediately

    # Save progress checkpoint
    save_progress(page_num, raw_text)

Agent Loop Design

Not a hardcoded pipeline — Claude Opus 4.5 decides at every step what to read, what to flag, what to escalate, and when to stop.

The Loop

Step 1 — PLAN
Call list_patient_pages → see all docs and types. Agent decides which to read first.
Steps 2-4 — READ
Call read_pages (max 5 at a time). Reads clinical notes, labs, medication charts in priority order.
Step 5-6 — RECONCILE
Extract admission meds + discharge meds → call reconcile_medications → formal diff.
Step 7-8 — CHECK & FLAG
Call check_drug_interaction, detect_conflict, escalate_to_clinician, flag_missing_field.
Step 9-10 — DRAFT
Produce final JSON summary. Every field either sourced with page citation or explicitly marked.

Hard Constraints

Step Cap

All 7 Agent Tools

Claude Opus 4.5 decides when to call each tool. Tools are defined with Anthropic's input_schema format and executed server-side with retry logic.

list_patient_pages(patient_id)
Returns a page index: page number, doc type (ADMISSION_NOTE, LAB_RESULT, MEDICATION_CHART, etc.), and 200-char preview. Called first on every run.
When: Always — Step 1 of every agent run
read_pages(patient_id, page_numbers)
Returns full OCR text for up to 5 pages. Max 5 per call to stay within Claude's context window. Agent calls multiple times for different document types.
When: After listing pages — reads clinical notes, labs, med charts in order of importance
reconcile_medications(admission_meds, discharge_meds)
Formal diff: compares two medication lists by name, flags added/stopped/changed/unchanged. Any unexplained change triggers a flag automatically.
When: After extracting both medication lists separately
detect_conflict(patient_id, field, source_a, value_a, source_b, value_b)
Records a clinical data conflict between two source documents. Writes to conflicts.json. Never resolves — always escalates to clinician.
When: When same field has different values in different documents
check_drug_interaction(medications)
Mock drug interaction DB with 9 known pairs (metformin+contrast, warfarin+aspirin, etc.). Simulates 5% timeout rate to test failure handling. Any hit → escalate.
When: After extracting discharge medication list
escalate_to_clinician(patient_id, field, reason)
Writes escalation to escalations.json with timestamp. Used for: drug interactions, undocumented medication changes, data conflicts, safety concerns.
When: Any safety concern, conflict, or unexplained clinical finding
flag_missing_field(patient_id, field_name, context)
Marks a required discharge summary field as MISSING or PENDING. Provides context about why it's missing (not in documents, OCR failure, etc.).
When: Every required field not found in source documents
No-Fabrication Guardrail

The #1 clinical safety requirement. Every field in the output is either explicitly sourced from a document, or explicitly marked as missing/pending/conflicted. The agent never guesses.

Five Field States

✓ FOUND Value extracted + source page cited
— MISSING Not found in any document
⏳ PENDING Result not yet available
⚡ CONFLICT Two docs disagree — both shown
🚨 FLAGGED Safety concern escalated
🔒The system prompt uses the phrase "NEVER invent, guess, or infer any clinical fact not explicitly stated in source documents" as an absolute rule, not a suggestion.

Pydantic Model

class ClinicalField(BaseModel):
    value: Optional[str] = None
    status: FieldStatus  # FOUND/MISSING/etc
    source: Optional[str] = None  # page X
    flag_reason: Optional[str] = None

    def display(self) -> str:
        if self.status == FOUND:
            return f"{self.value} [src:{self.source}]"
        elif self.status == MISSING:
            return "[MISSING — clinician review]"
        elif self.status == CONFLICT:
            return f"[CONFLICT: {self.flag_reason}]"
✅The parse_field() function handles ANY type Claude returns (str, list, dict, None) — it never crashes, always falls back to MISSING status.

Output always labeled

Every discharge summary starts with: "THIS IS A DRAFT FOR CLINICIAN REVIEW — NOT FINAL" — enforced in the model's to_readable() method, not optional.

Medication Reconciliation

Formal comparison of what medications a patient came in on vs what they're discharged with. Every unexplained change is flagged.

reconcile_medications() Logic

def reconcile_medications(
    admission_meds: list,
    discharge_meds: list
) -> dict:
    adm = {m['name'].lower(): m for m in admission_meds}
    dis = {m['name'].lower(): m for m in discharge_meds}

    # In discharge but not admission = ADDED
    for name in dis - adm:
        if not has_reason(dis[name]):
            flags.append(f"ADDED with no reason: {name}")

    # In admission but not discharge = STOPPED
    for name in adm - dis:
        flags.append(f"STOPPED — no documentation: {name}")

    # Dose changed = CHANGED
    for name in adm & dis:
        if adm[name]['dose'] != dis[name]['dose']:
            flags.append(f"DOSE CHANGED: {name}")

Part 2 — Learning Loop Design

The assignment says to manufacture a feedback loop. We built: Simulated Doctor → Extract Corrections → Store in Pinecone → Inject into Future Runs.

The Architecture

Step 1 — Agent generates draft (iter 0)
No corrections in Pinecone yet. Baseline quality.
Step 2 — Simulated Doctor Reviews
Claude Opus 4.5 plays a senior clinician with a hidden editing policy. Applies consistent formatting and clinical corrections.
Step 3 — Extract Correction Patterns
Claude compares draft vs edited version, extracts generalizable patterns. e.g. "Always expand 1-0-0 to once daily (morning)"
Step 4 — Store in Pinecone
Each pattern embedded with Jina AI (1024-dim) and stored permanently. Never lost — survives restarts.
Step 5 — Next run (iter 1+)
Query Pinecone for relevant corrections → inject top 8 into system prompt → better draft → lower edit burden.

Doctor's Hidden Policy

Agent does NOT see these rules. Must learn from corrections:

Medication format
1-0-0 → "once daily (morning)" · BD → "twice daily"
Always add route
oral / IV / subcutaneous / topical always specified
Diagnosis format
"Condition — qualifier" e.g. "DKA — resolved on discharge"
Follow-up standard
Always ends: "Return to ED if symptoms worsen"
Pending results note
Always adds: "Results to be communicated by treating team/GP"
💡Approach: Structured correction-memory injected into future prompts — one of the valid approaches listed in the assignment. Chosen because it's explainable, measurable, and works without fine-tuning access.
Pinecone + Jina AI — Memory Architecture

Cloud-native, never loses data. 61 correction vectors stored permanently. Semantic retrieval finds relevant past corrections for any new patient.

Embedding + Retrieval Flow

# STORE a correction
text = f"Section: {section}. Pattern: {pattern}"
embedding = jina_embed([text])[0]  # 1024-dim
pinecone.upsert(vector=embedding, metadata=correction)

# RETRIEVE for new patient
query = f"Section: discharge_medications. Context: DKA"
q_emb = jina_embed([query])[0]
results = pinecone.query(
    vector=q_emb, top_k=5,
    filter={"score": {"$gt": 0.3}}
)

# INJECT into agent prompt
corrections_text = format_for_prompt(results)
system_prompt = BASE_PROMPT + corrections_text

Improvement Metrics

Section-level normalized edit distance measures how much a doctor had to change the draft. Lower edit burden = higher reward = agent learning.

Reward Formula

from difflib import SequenceMatcher

def similarity(draft, edited):
    return SequenceMatcher(
        None, draft.lower(), edited.lower()
    ).ratio()
    # 0.0 = completely rewritten
    # 1.0 = no edits needed

# Weighted average across sections
weights = {

What worked

Limitations found
